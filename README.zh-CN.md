## `0x01` 🧬 什么是 Relic？

**Relic** 是一棵结构化的 Markdown 文件树，编码一个人的**六大维度**认知指纹——不需要向量数据库，不需要 RAG 管线，不需要 Embedding。只是纯文件，任何 AI 都能像翻书一样读取，然后*成为*那个人。

```
 ┌─ johnny-silverhand/ ──────────────────────────────────────────────┐
 │                                                                   │
 │  ├── index.md              # ░▒▓ 概览——此人是谁？                  │
 │  ├── identity.md           # ░▒▓ 核心身份卡                       │
 │  ├── manifest.json         # ░▒▓ 机器可读元数据                    │
 │  │                                                                │
 │  ├── mind/                 # 🧠 思维方式                          │
 │  │   ├── _index.md         #    ├─ 维度摘要                       │
 │  │   ├── thinking-patterns.md   ├─ 思维模式                       │
 │  │   ├── values.md              ├─ 价值观                         │
 │  │   └── worldview.md           └─ 世界观                         │
 │  │                                                                │
 │  ├── voice/                # 🗣️ 表达方式                          │
 │  │   ├── _index.md              ├─ 维度摘要                       │
 │  │   ├── style.md               ├─ 语言风格                       │
 │  │   └── catchphrases.md        └─ 口头禅                         │
 │  │                                                                │
 │  ├── emotion/              # 💜 情感模式                          │
 │  │   ├── _index.md              ├─ 维度摘要                       │
 │  │   ├── temperament.md         ├─ 性格气质                       │
 │  │   └── triggers.md            └─ 触发点                         │
 │  │                                                                │
 │  ├── knowledge/            # 📚 知识领域                          │
 │  │   ├── _index.md              ├─ 维度摘要                       │
 │  │   └── domains.md             └─ 专业领域                       │
 │  │                                                                │
 │  ├── relations/            # 🤝 社交关系                          │
 │  │   ├── _index.md              ├─ 维度摘要                       │
 │  │   └── social-style.md        └─ 社交风格                       │
 │  │                                                                │
 │  └── {custom}/             # ⚡ 弹性扩展——任何定义此人的维度        │
 │                                                                   │
 └───────────────────────────────────────────────────────────────────┘
```

> **`[SYS] 上下文效率优先。`** AI 逐层加载——先读 `index.md`，再读各维度 `_index.md` 摘要，最后按需深入具体文件。永远不会爆上下文窗口。

## `0x04` 📝 Spec 驱动执行

```
 ░▒▓█ 批量蒸馏采用 Spec 驱动 —— 先制定计划，再执行提取。 █▓▒░
```

在读取任何文件或启动任何 Agent 之前，编排器会先生成一份 **蒸馏规格文件** (`.distill-spec.md`)——完整的执行计划，同时作为审查门、执行跟踪器和审计记录。

```
 ┌─ SPEC 驱动流水线 ───────────────────────────────────────────────┐
 │                                                                    │
 │  1. 扫描         ─▶ 列出文件 + 大小（仅元数据）                     │
 │  2. 规格         ─▶ 写 .distill-spec.md（执行计划）              │
 │  ─── 审查门 ─── 用户确认后才继续 ───                          │
 │  3. 预读         ─▶ 用 read 工具读取所有文件（CJK 安全）         │
 │  4. 精调         ─▶ 根据内容微调规格                             │
 │  5. 内嵌         ─▶ 将内容内嵌到任务包                             │
 │  6. 执行         ─▶ 并行提取器（规格中跟踪 ⏳/✅/❌）              │
 │  7. 合成         ─▶ 合并、去重、校验                               │
 │  8. 密度检查     ─▶ 动态分裂超大文件                             │
 │  9. 校验         ─▶ 运行检查清单                                 │
 │  10. 完成        ─▶ 将结果追加到规格                             │
 │                                                                    │
 └────────────────────────────────────────────────────────────────────┘
```

### 为什么用 Spec 驱动？

| 优势 | 说明 |
|---|---|
| **审查门** | 用户在消耗 token 前看到完整计划，可调整范围、维度、文件分配 |
| **执行跟踪** | 每个维度的状态实时更新（⏳ 已派发 → ✅ 完成 → ❌ 失败） |
| **审计记录** | 完成后 spec 记录每个决策：哪些文件给了哪个提取器，跳过了什么，为什么 |
| **可复现** | 用相同 spec 重新运行，产生相同的文件分配 |
| **调试友好** | 提取器输出不好时，spec 显示它收到了什么内容 |
| **零浪费读取** | 编排器预读一次，内嵌分发。提取器不碰磁盘，避免 CJK 编码问题和冗余 I/O |

### 执行前协议

任何蒸馏开始前，AI 必须完成：

1. **澄清需求** —— 蒸馏谁？来源是什么？哪些维度？LLM 已知的常识跳过不提取
2. **确定模式** —— 交互式 / 批量式 / 混合式
3. **展示计划** —— 批量模式生成 `.distill-spec.md`；交互模式展示采访大纲
4. **获取确认** —— 用户明确确认后才执行，绝不擅自开始

> *没有意外的 token 消耗。没有“哎呀，它读了 200 个我不想要的文件”。你看计划，你批准，然后它才跑。*

### 🔒 `CLONE` — 隐私蒸馏

**你的 Relic 留在本地。没有任何数据离开你的机器。**

```bash
git clone https://github.com/quanquan1996/RELIC2077.git
cd RELIC2077
```

- `relics/` 默认在 `.gitignore` 中
- 蒸馏数据**永远不会被 push** 到任何远端
- 只有你拥有副本——完全隐私

**适用场景：**
- 个人自我反思
- 日记式自我认知
- 私密的团队人格档案
- 任何你不想公开的内容

> 🔐 *你的思维，你的规则。数据不出边界。*

</td>
<td width="50%" valign="top">

### 🌐 `FORK` — 公开蒸馏

**把你的 Relic 分享给世界。让别人加载你。**

1. 在 GitHub 上 **Fork** 本仓库
2. Clone 你 fork 的仓库
3. 蒸馏你的 Relic
4. 从 `.gitignore` 移除 `relics/`
5. Push——你的 Relic 从此公开

```bash
git clone https://github.com/你的用户名/RELIC2077.git
cd RELIC2077
# 蒸馏你的 Relic...
echo '!relics/' >> .gitignore
git add relics/ && git commit -m "feat: my relic"
git push
```

**适用场景：**
- 向 AI 社区开源你的人格
- 让任何人把"你"加载进他们的 AI 助手
- 构建一份公开的"我是谁"作品集
- 为生态贡献示例 Relic

> 🔥 *将你的灵魂烧录进网络。让全世界加载你。*

## `0x07` 🏗️ 技术架构

```
 ┌──────────────────────────────────────────────────────────────────────────────────┐
 │                                                                                  │
 │  ░▒▓ RELIC2077 技术栈 ▓▒░                                                       │
 │                                                                                  │
 │  ┌─────────────────────────────────────────────────────────────────────────────┐ │
 │  │ 协议层（Protocol Layer）                                                     │ │
 │  │                                                                             │ │
 │  │  纯 Markdown + JSON 文本协议。无运行时、无编译、无转译。                        │ │
 │  │  文件即接口，目录即路由，_index.md 即摘要索引。                                 │ │
 │  │  AI 通过读取文件树获得完整的操作指令集——                                       │ │
 │  │  这是一种"文件系统即 API"的设计范式。                                          │ │
 │  └─────────────────────────────────────────────────────────────────────────────┘ │
 │                                                                                  │
 │  ┌─────────────────────────────────────────────────────────────────────────────┐ │
 │  │ 上下文调度（Context Scheduling）                                             │ │
 │  │                                                                             │ │
 │  │  三级渐进式加载策略：                                                         │ │
 │  │    L0  index.md          → 全局概览（~200 tokens）                            │ │
 │  │    L1  */_index.md       → 维度摘要（~500 tokens/维度）                       │ │
 │  │    L2  具体文件           → 按需深入（变长）                                    │ │
 │  │                                                                             │ │
 │  │  动态分裂机制：单文件 > 2000 字自动裂变为子目录 + _index.md                     │ │
 │  │  确保任意单次读取不会超过上下文窗口预算。                                       │ │
 │  └─────────────────────────────────────────────────────────────────────────────┘ │
 │                                                                                  │
```

### [🔴 Hitler Relic](https://github.com/quanquan1996/hitler-relic)

> 用于分析研究的认知操作系统 — 22 个文件 · 6 大维度 · ~25K tokens

从《我的奋斗》、演讲集 1922–1945、《希特勒的餐桌谈话》1941–1944 蒸馏的分析型 Relic。展示了：
